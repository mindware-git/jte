# 동유기 변경 파트 설계 메모

이 폴더는 기존 [parts](/Users/kwang/Documents/git-repos/jte/docs/story/parts) 구조를 동유기 버전으로 다시 짠 세부 시나리오 초안이다.

핵심 원칙은 다섯 가지다.

- 기존 환상서유기의 `분기 구조`, `숨겨진 아이템 조사`, `기믹 던전`, `반복 대화 보상`을 적극 재활용한다.
- 스토리 톤은 정극보다 `해학`, `꽁트`, `민담풍 말장난`에 무게를 둔다.
- 아이템, 스킬, UI/UX는 원본 이름을 억지로 유지하지 않는다. `기능은 참고하되 이름과 맥락은 동유기답게 재정의`한다.
- 각 파트는 `스토리`, `퍼즐`, `개그`, `재활용 시스템`을 같이 설계한다.
- `이 장소에 꼭 그 기믹이 있어야 한다`는 식의 직접 대응은 피한다. 재미가 더 살아난다면 장소, 동선, 목표물, 퀘스트 구조는 과감하게 바꾼다.

문서 읽는 법:

- `줄거리`: 파트의 큰 사건
- `퍼즐/탐색`: 미로, 기믹, 조사 요소
- `숨은 요소`: 반복 대화, 벽 조사, 조건부 아이템 등
- `꽁트 포인트`: 컷신/필드 대사에서 웃길 지점
- `시스템 재활용`: 원작의 어떤 감각을 가져오는지

이 문서들은 완성 대본이 아니라 `인게임 구현용 설계서`에 가깝다.

추가 메모:

- 원본의 `아이템 기능`이 그대로 필요하더라도 이름은 `청심환`, `해롱약`, `도깨비 호루라기`, `풍륜심목`, `안개부적`처럼 새로 붙여도 된다.
- 원본의 `스킬 역할`이 유지되면, 표현은 `홀리 워터`보다 `여우불`, `정화염`, `도깨비 잔재주`, `해치의 법화`, `카게베기`처럼 동유기식으로 바꿔도 된다.
- 플레이어가 기억해야 할 것은 원본 이름이 아니라 `재미있는 상황`, `찾는 재미`, `다음에 뭘 해 볼까 하는 호기심`이다.
